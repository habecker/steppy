Lightweight workflow engine

Steppy lets you compose small, reusable Java steps into workflows called flows. Each step declares its input and output types and Steppy wires them together for sequential or concurrent processing.

Key ideas:

  • Step – a unit of work with typed input and output.
  • Flow – a pipeline executing steps.
  • FlowBuilder – a fluent API to assemble flows.
  • StepRepository – pluggable lookup for step instances.

Explore the chapters below to see Steppy in action:

  • Getting started – build a small workflow.
  • Flows – run steps sequentially or concurrently.
  • Branching & Nesting – route and compose subflows.
  • Streaming – work with infinite sources.
  • State & Lifecycle – share variables and hook into execution.

  • Dependency injection – integrate with DI frameworks.
  • Validation & Errors – type checking and error handling.
